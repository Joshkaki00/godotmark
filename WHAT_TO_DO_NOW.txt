PROBLEM:
--------
You tried to run GodotMark on your Raspberry Pi 5, but got this error:

  ERROR: GDExtension dynamic library not found: 'res://godotmark.gdextension'.

WHY:
----
The C++ GDExtension library was built for Windows (x86_64), not ARM64 Linux.
Your RPi5 needs an ARM64 Linux version of the library.

SOLUTION:
---------
Build the library NATIVELY on your Raspberry Pi 5.

================================================================================
                             üöÄ DO THIS NOW
================================================================================

STEP 1: Open Terminal on RPi5
------------------------------

STEP 2: Navigate to Project
----------------------------
cd /mnt/exfat_drive/dev/godotmark-project/godotmark

STEP 3: Make Build Script Executable
-------------------------------------
chmod +x build_native_rpi5.sh

STEP 4: Build the Library (THIS WILL TAKE 10-20 MINUTES)
---------------------------------------------------------
./build_native_rpi5.sh template_release rpi5 yes

WHAT THIS DOES:
  ‚úÖ Checks for dependencies (scons, g++, python3)
  ‚úÖ Installs them if missing
  ‚úÖ Initializes godot-cpp submodule
  ‚úÖ Compiles C++ code optimized for your RPi5
  ‚úÖ Creates: bin/libgodotmark.linux.template_release.arm64.so

STEP 5: Verify Build
--------------------
chmod +x check_build.sh
./check_build.sh

YOU SHOULD SEE:
  ‚úÖ Release library found:

================================================================================
                        ‚ö†Ô∏è TROUBLESHOOTING
================================================================================

IF BUILD FAILS: "scons: command not found"
------------------------------------------
sudo apt update
sudo apt install -y scons build-essential python3 git

IF BUILD FAILS: "godot-cpp not found"
--------------------------------------
cd /mnt/exfat_drive/dev/godotmark-project/godotmark
git submodule update --init --recursive

IF BUILD FAILS: "Out of memory" or "Killed"
--------------------------------------------
# Reduce parallel jobs (use -j2 instead of -j4)
scons platform=linux arch=arm64 target=template_release cpu=rpi5 optimize_size=yes -j2

# OR enable more swap
sudo dphys-swapfile swapoff
sudo nano /etc/dphys-swapfile  # Change CONF_SWAPSIZE to 2048
sudo dphys-swapfile setup
sudo dphys-swapfile swapon

IF STILL GETTING "GDExtension not found" AFTER BUILD
-----------------------------------------------------
1. Check the library exists:
   ls -lh bin/libgodotmark.linux.template_release.arm64.so

2. Check it's ARM64:
   file bin/libgodotmark.linux.template_release.arm64.so
   (should say "ARM aarch64")

3. Try running from correct directory:
   cd /mnt/exfat_drive/dev/godotmark-project
   ./Godot_v4.4-stable_linux.arm64 --path godotmark
